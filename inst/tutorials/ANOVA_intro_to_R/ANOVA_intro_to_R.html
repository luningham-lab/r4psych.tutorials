Download R and RStudio

Before we begin, make sure you have both R and RStudio installed:

  1. Download R from the Comprehensive R Archive Network: https://cran.r-project.org/
  2. Download RStudio Desktop from Posit: https://posit.co/download/rstudio-desktop/

R is the engine. RStudio is the “integrated development environment” (IDE), with a graphical user interface, code help functions, workspace management, and data viewer tools.

The RStudio Environment

RStudio includes 4 main windows (panes):

  • Script editor for writing, debugging, and saving code
  • Console where input and output are shown; code can also be typed here directly
  • Environment pane, which shows available data, saved objects, functions, and script history (also includes a point-and-click Import Dataset button)
  • Utility pane, which can show plots, tables, help windows, file paths, and a Tutorial tab

Working in R

There are a few ways to execute code in RStudio:

  1. Type directly into the Console after the > prompt, and hit Enter
  2. Write code in the Script editor, highlight lines, and run them (Run button, Cmd+Return on Mac, Ctrl+Enter on Windows)
  3. More advanced: call and run an existing saved script from within a current script

The script editor approach is preferred because syntax can be edited, reused, and — importantly — saved for future replication.

R Basics: Math and Objects

R as a calculator

Below is your first view of a code chunk. This is equivalent to putting code into an R script and then running it or writing code in the console and hitting enter. These are built in to our learning modules for practice.

In the code chunk below, type 8 + 5 and click Run Code.

8 + 5

The result is printed, but it’s not saved anywhere. If we wanted to use the number 13 again later, we’d have to retype the calculation. Instead, we can store it in an object.

The Assignment Operator

To create an object, we use the assignment operator, <-. Assignments have the form:

object <- value

We read this as: “the object gets this value.”

Run the code below. The number 13 is now stored in the R environment under the name thirteen.

thirteen <- 13
thirteen

Assignments can also store the result of a function. Here the function is simply the + operation:

thirteen <- 8 + 5
thirteen

Once stored, we can use objects in future operations:

thirteen <- 8 + 5
five <- 5
thirteen + five

Data Types

We can assign anything to an object, not just numbers. For example, we can store text (called a character string) by using quotation marks:

thirteen <- "thirteen"
thirteen

However, R is very sensitive to the type of element. The most common types are:

  • numeric (also called double or dbl) — numbers
  • character (or chr) — text strings
  • factor (or fct) — categorical labels

What happens if we try to add a character to a number? Try running the code below:

thirteen <- "thirteen"
five <- 5
thirteen + five

You’ll get an error! The str() function reveals the structure of an object, which helps diagnose these issues:

thirteen <- "thirteen"
five <- 5
str(thirteen)
str(five)

Functions

What is a function?

Functions are pre-built commands in R that perform specific operations. They have the general form:

functionName(arg1 = val1, arg2 = val2, ...)

Arguments are provided within the parentheses. They tell the function what to operate on and how.

We already saw str(), which displays the structure of any R object. Another example is mean(), which calculates the arithmetic mean. It requires an R object such as a numeric vector.

To create a vector (a sequence of values), we use the c() function, which combines values:

mean(x = c(8, 5))

Run the code below to see another useful function — summary() — applied to a vector:

my_data <- c(3, 7, 8, 5, 12, 10, 6)
summary(my_data)

Getting Help

You can get help about any function by typing ? followed by the function name in the console (e.g., ?mean). In RStudio, the function’s documentation will appear in the Help window of the utility pane.

Documentation typically includes:

  • A description of what the function does
  • What arguments it requires
  • What output it produces
  • Examples

R Packages

Installing and Loading Packages

R is open-source and has thousands of contributed packages on CRAN. Packages extend R’s functionality for specific tasks.

To use a package, two steps are required:

  1. Install the package (only once, or when updating):
install.packages("dplyr")
  1. Load it into the active library (every R session):
library(dplyr)

Notice the difference: install.packages() requires the package name in quotes (a character string), while library() does not require quotes.

Data Manipulation with dplyr

Introduction to dplyr

The dplyr package (the d is for “data frames”, plyr is a reference to “pliers”) provides a suite of intuitive functions for data manipulation. There are 5 core functions:

  • select() — picks variables (columns) based on their names
  • filter() — picks cases (rows) based on their values
  • summarize() — reduces multiple values down to a single summary
  • mutate() — adds new variables that are functions of existing variables
  • arrange() — changes the ordering of rows

The Pipe Operator

A unique and powerful feature of dplyr is the pipe operator: |>. Think of it as saying “and then…”

The pipe operator was first introduced as part of the tidyverse, but it is now built into base R (called “native pipe”). It can be inserted with Ctrl + Shift + M.

If you have learned to use the pipe before, you may have seen it as %>%. This is the tidyverse pipe. The native pipe |> is now preferred because its use does not depend on loading specific libraries. However, in practice, both will work if you load dplyr.

We specify a dataset, then pipe it through one or more operations:

data |> 
  do_something() |> 
  do_something_else()

select() — Pick Columns

library(dplyr)
library(AMCP)
data(chapter_1_table_1)
data(chapter_3_table_3)
chapter_3_table_3 <- chapter_3_table_3 |> mutate(Condition = as.factor(Condition))

select() chooses which columns to keep. Run the examples below:

chapter_1_table_1 |> 
  select(treat)

chapter_1_table_1 |> 
  select(week, treat, control)

filter() — Pick Rows

filter() keeps only rows that meet a condition:

chapter_1_table_1 |> 
  filter(week == 1)

Note the double equals == for logical comparison. A single = is used for assignment, like <-, but == directly compares if two objects or numbers are the same. It returns a TRUE or a FALSE. filter keeps only the rows where the condition is TRUE.

summarize() — Compute Summaries

summarize() collapses a dataset down to summary statistics. You provide specific functions (like mean(), sd()) inside it.

The code below is multi-part: using the full data set, we then filter to only Week 1 observations, then we create a summary table containing the mean, sd, and total sum of the treatment group column.

chapter_1_table_1 |> 
  filter(week == 1) |> 
  summarize(
    Mean_treat = mean(treat), 
    SD_treat = sd(treat), 
    Sum_treat = sum(treat)
  )

group_by() + summarize() — The Power Combo

The group_by() function tells dplyr to perform subsequent operations separately for each group. This is extremely useful for ANOVA contexts:

chapter_1_table_1 |> 
  group_by(week) |> 
  summarize(
    Mean_treat = mean(treat), 
    SD_treat = sd(treat)
  )

mutate() — Create New Variables

mutate() adds new columns to the data frame. The function form is mutate(new_column_name = some_function(of_existing + columns):

chapter_1_table_1 |> 
  mutate(Difference = treat - control)

Important: The mutated data above is only printed, not saved. To save changes, assign the result back to an object:

chapter_1_table_1 <- chapter_1_table_1 |> 
  mutate(Difference = treat - control)

count() — Quick Frequency Tables

count() is a handy shortcut for tallying up frequencies of categorical variables:

chapter_3_table_3 |> 
  count(Condition)

Visualizing Data

Base R Plots

R has many built-in functions for making simple plots. The boxplot() function requires a formula of the form dependent_variable ~ grouping_variable:

boxplot(Rating ~ Condition,
        data = chapter_3_table_3,
        ylab = "Global Affect Rating", 
        xlab = "Group",
        main = "Box and Whiskers Plot of Affect Rating by Group",
        col = "bisque",
        names = c("Pleasant", "Neutral", "Unpleasant"))

Histograms

A histogram shows the distribution of a single variable. One simple function for a histogram is simply called hist(). The lines function adds lines on an existing plot, and density will create a smooth probability curve on top of the histogram bars:

hist(chapter_3_table_3$Rating, #provide the single column of data
     probability = TRUE, #y-axis as counts or proportions
     main = "Histogram and Estimated Density Line") #main = "Title"
lines(density(chapter_3_table_3$Rating))

Introduction to ggplot2

The ggplot2 package can produce elegant, publication-quality plots. It has some unique conventions:

  • ggplot() initializes a plot, specifying the data and “aesthetics” (aes()) — what variable maps to x, y, color, etc.
  • Layers are added with + (similar in spirit to the |> pipe)
  • Different plot types are called geoms (e.g., geom_histogram(), geom_boxplot())
  • facet_wrap() splits a plot by levels of a categorical variable
  • theme_bw() provides a clean black-and-white theme
ggplot(chapter_3_table_3, aes(x = Rating)) +
  geom_histogram() +
  facet_wrap(~ Condition) +
  theme_bw(base_size = 16)

The aov() Function

The basic R function to compute an ANOVA is aov(). It requires a formula (dependent_var ~ independent_var) and the dataset as primary arguments.

We typically save the results of the aov() function through assignment, and then we can use a helper function called summary() to view the results.

Important: The independent variable must be a factor for aov() to work correctly. I’ve already done this, but the commented-out code below shows how.

  #mutate(Condition = as.factor(Condition))

Computing ANOVA Step by Step

We can compute the elements of an ANOVA by hand in R. First, let’s set up placeholders:

# Number of groups and total N
a <- length(unique(chapter_3_table_3$Condition))
N <- nrow(chapter_3_table_3)

# Full model errors (sum of within-group SS)
Summary.Mood <- chapter_3_table_3 |> 
  group_by(Condition) |>
  summarize(SSE = sum((Rating - mean(Rating))^2))

E.F <- sum(Summary.Mood$SSE)
df.F <- N - a

# Restricted model errors (total SS from grand mean)
E.R <- sum((chapter_3_table_3$Rating - mean(chapter_3_table_3$Rating))^2)
df.R <- N - 1

# F statistic
F_stat <- ((E.R - E.F) / (df.R - df.F)) / (E.F / df.F)

cat("F =", F_stat, "\n")
cat("Critical value =", qf(.95, df.R - df.F, df.F), "\n")
cat("p-value =", 1 - pf(F_stat, df.R - df.F, df.F), "\n")

A Common Pitfall: Factor Variables

A very common mistake is forgetting to convert grouping variables to factors. Look what happens with the alcoholism treatment data:

data(chapter_3_table_7_raw)
str(chapter_3_table_7_raw)

# Group is numeric, not factor --- this gives wrong results!
mdl_wrong <- aov(Drinks ~ Group, data = chapter_3_table_7_raw)
summary(mdl_wrong)

Notice there’s only 1 degree of freedom for Group — R is treating it as a continuous number! The fix:

data(chapter_3_table_7_raw)

chapter_3_table_7_raw <- chapter_3_table_7_raw |> 
  mutate(Group = as.factor(Group))

mdl_correct <- aov(Drinks ~ Group, data = chapter_3_table_7_raw)
summary(mdl_correct)

Now we get the correct degrees of freedom for 5 groups.

Checking Assumptions

Why Check Assumptions?

Before trusting ANOVA results, we should verify:

  1. The dependent variable is approximately normally distributed within groups
  2. Groups have approximately equal variances (homogeneity of variance)

Visualizations are the first step.

Histograms by Group

Use ggplot2 with facet_wrap() to check normality within each group:

ggplot(chapter_3_table_3, aes(x = Rating)) +
  geom_histogram(bins = 6) +
  facet_wrap(~ Condition) +
  theme_bw(base_size = 16)

Means and Variability

We can use the gplots package’s plotmeans() function to visualize group means with confidence intervals, which helps assess equal variance:

library(gplots)
plotmeans(Rating ~ Condition, 
          data = chapter_3_table_3, 
          connect = FALSE,
          main = "Confidence Interval Plot")
