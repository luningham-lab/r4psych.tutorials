A Motivating Example

The Hypertension Data

Table 4.1 from the text presents hypothetical data for four groups of participants receiving different hypertension treatments, measured on systolic blood pressure. The four treatment groups are: Drug Therapy, Biofeedback, Diet, and Combination.

Let’s start by loading and preparing the data. We will use case_match() from dplyr to recode the numeric condition variable into meaningful labels. Recall from the first tutorial that mutate() adds or modifies columns in a data frame, and that grouping variables must be converted to factors for ANOVA.

library(dplyr)
library(AMCP)

data(C4T1)

head(C4T1)
str(C4T1)

Now let’s recode the conditions and convert to factors. The case_match() function takes the variable to recode, then a series of formulas: old_value ~ "new_label". We wrap this inside mutate() to create the new variable:

C4T1 <- C4T1 |> 
  mutate(group = case_match(cond,
    1 ~ "Drug Therapy",
    2 ~ "Biofeedback",
    3 ~ "Diet",
    4 ~ "Combination"
  )) |>
  mutate(group = factor(group), cond = factor(cond))

str(C4T1)

Notice the use of the native pipe |>. This works the same as the %>% pipe from the first tutorial — it passes the result on the left into the first argument of the function on the right. The |> pipe is built into base R (version 4.1+), so it does not require any additional packages.

The Omnibus ANOVA

Running aov()

As we learned in the first tutorial, aov() requires a formula (dependent ~ independent) and the dataset. Let’s run the omnibus one-way ANOVA:

mdl <- aov(bloodpr ~ group, data = C4T1)
summary(mdl)

The omnibus F-test tells us whether there is any significant difference among the four group means. If the p-value is below our alpha level, we reject the null hypothesis that all group means are equal.

But the omnibus test doesn’t tell us where the differences are. For that, we need follow-up comparisons — and this is where emmeans comes in.

Introduction to emmeans

What is emmeans?

The emmeans package (Estimated Marginal Means) is a powerful and flexible tool for conducting follow-up comparisons after fitting a model. It can handle:

  • Pairwise comparisons between all group means
  • Custom (complex) contrasts that test specific hypotheses
  • Adjustments for multiple comparisons (e.g., Tukey, Bonferroni)
  • Comparisons in more complex designs (factorial, repeated measures — we will cover these later)

The basic workflow is:

  1. Fit a model with aov() (or lm())
  2. Extract the estimated marginal means with emmeans()
  3. Conduct comparisons with pairs() or contrast()

To install emmeans (if you haven’t already):

install.packages("emmeans")
library(emmeans)

Estimated Marginal Means

The first step after fitting a model is to extract the estimated marginal means (EMMs). These are the model-based estimates of each group mean.

The emmeans() function requires an already fitted model as its argument. In other words, we assign the aov() result to an object, then provide that object to emmeans(). It recognizes the original data and the variable names for the groups.

To get the groups, we ask for part of the formula again, with ~ group_var_name within in the function.

mdl <- aov(bloodpr ~ group, data = C4T1)
emm <- emmeans(mdl, ~ group)
emm

The output shows each group’s estimated mean, the standard error, degrees of freedom, and a confidence interval. In a balanced one-way design, these estimated means are identical to the simple group means we computed with summarize(). The advantage of emmeans becomes clear in more complex or unbalanced designs, and — as we’ll see next — for conducting comparisons.

Pairwise Comparisons

All Pairwise Tests

The simplest follow-up after a significant omnibus ANOVA is to compare every pair of group means. The pairs() function does this automatically:

mdl <- aov(bloodpr ~ group, data = C4T1)
emm <- emmeans(mdl, ~ group)
pairs(emm)

By default, pairs() applies a Tukey adjustment for multiple comparisons, which controls the familywise error rate across all pairwise tests. The output shows the estimated difference (estimate), standard error, degrees of freedom, t-ratio, and the adjusted p-value for each pair.

Changing the Adjustment Method

You can change the p-value adjustment method with the adjust argument. Common options include "tukey" (default), "bonferroni", "sheffe", "dunnettx", "fdr" (Benjamini-Hochberg), and "none":

mdl <- aov(bloodpr ~ group, data = C4T1)
emm <- emmeans(mdl, ~ group)

# No adjustment (unadjusted p-values)
pairs(emm, adjust = "none")

mdl <- aov(bloodpr ~ group, data = C4T1)
emm <- emmeans(mdl, ~ group)

# Bonferroni adjustment
pairs(emm, adjust = "bonferroni")

Custom (Complex) Contrasts

Beyond Pairwise Comparisons

Pairwise comparisons test every possible pair, but sometimes we have specific, theory-driven hypotheses. For example:

  • Does the Combination treatment differ from the average of the other three treatments?
  • Do the non-drug treatments (Biofeedback, Diet, Combination) differ from Drug Therapy alone?

These are complex contrasts — comparisons that involve more than two groups at a time. The emmeans package handles these with the contrast() function.

Setting Up Custom Contrasts

To define a custom contrast, we create a named list of coefficient vectors. Each vector must have one value per group, and the coefficients must sum to zero (the definition of a contrast).

First, let’s check the order of the groups in our emmeans object, since the coefficients must match this order:

mdl <- aov(bloodpr ~ group, data = C4T1)
emm <- emmeans(mdl, ~ group)
emm

The groups are listed in the output. By default, R orders factor levels alphabetically, so the order here is: Biofeedback, Combination, Diet, Drug Therapy. This is the order your coefficients must follow.

Now let’s test the hypothesis that the Combination treatment differs from the average of the other three. The contrast coefficients for this comparison are:

Group Biofeedback Combination Diet Drug Therapy
Coefficient 1/3 -1 1/3 1/3

These sum to zero (1/3 + (-1) + 1/3 + 1/3 = 0), confirming this is a valid contrast.

emm <- emmeans(mdl, ~ group)

# Combination vs. average of all others
combo_vs_rest <- list("Combo vs Others" = c(1/3, -1, 1/3, 1/3))

contrast(emm, combo_vs_rest)

Testing Multiple Custom Contrasts

We can test multiple contrasts at once by adding more entries to the list. Let’s also test whether Drug Therapies (Drug + Combo) differ from the average of the non-drug treatments (Biofeedback, Diet):

emm <- emmeans(mdl, ~ group)

my_contrasts <- list(
  "Combo vs Others"       = c(1/3, -1, 1/3, 1/3),
  "Drug vs Non-Drug"      = c(-1/2, 1/2, -1/2, 1/2)
)

contrast(emm, my_contrasts)

You can also apply a p-value adjustment across your custom contrasts:

emm <- emmeans(mdl, ~ group)

my_contrasts <- list(
  "Combo vs Others"       = c(1/3, -1, 1/3, 1/3),
  "Drug vs Non-Drug"      = c(-1/2, 1/2, -1/2, 1/2)
)

contrast(emm, my_contrasts, adjust = "bonferroni")

This is helpful in case we have very specific pairwise comparisons, and we don’t want the multiple testing penalty of testing all pairs.

Confidence Intervals for Contrasts

You can obtain confidence intervals for any comparison by wrapping the result in confint():

emm <- emmeans(mdl, ~ group)

combo_vs_rest <- list("Combo vs Others" = c(1/3, -1, 1/3, 1/3))

confint(contrast(emm, combo_vs_rest))

Effect Sizes

The emmeans package can also provide standardized effect sizes for contrasts using eff_size(). This computes a Cohen’s d-type measure using the pooled within-group SD:

emm <- emmeans(mdl, ~ group)

# Effect sizes for pairwise comparisons
eff_size(emm, sigma = sigma(mdl), edf = df.residual(mdl))

The sigma(mdl) extracts the residual standard deviation (i.e., the square root of \(MS_W\)), and df.residual(mdl) provides the error degrees of freedom.

The emmeans Workflow

The general workflow for follow-up tests is:

  1. mdl <- aov(DV ~ IV, data = mydata) — fit the model
  2. emm <- emmeans(mdl, ~ IV) — extract estimated marginal means
  3. pairs(emm) — for pairwise comparisons, or
  4. contrast(emm, my_list) — for custom contrasts

We will continue to use emmeans in future tutorials as we move to factorial and repeated measures designs.
